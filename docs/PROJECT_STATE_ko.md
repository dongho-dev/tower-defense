# 프로젝트 스냅샷: 타워 디펜스

## 개요
- 단일 페이지 HTML5 캔버스 기반 타워 디펜스 게임 (`index.html`, `style.css`, `main.js`)
- 시작 자원: 골드 100, 라이프 20 (`main.js:46-52`)
- `update()` 루프가 웨이브를 자동으로 진행하며, 한 웨이브가 끝나면 약 4초 후 다음 웨이브가 시작됨 (`main.js:539-588`)

## 파일 구성
- `index.html:10-60` - HUD(웨이브/라이프/골드), 속도 제어, 웨이브 선택, 웨이브 미리보기, 포탑/적 정보 패널, 패배 오버레이
- `style.css:1-189` - 전체 레이아웃, 패널/오버레이/버튼 스타일, 공용 `.hidden` 클래스
- `main.js:1-808` - 상수 정의, 상태 관리, 입력 처리, 렌더링 및 게임 루프

## 조작 및 UI
- 빈 타일 좌클릭: 포탑 설치 (비용 25) (`main.js:420-445`)
- 포탑/적 좌클릭: 해당 정보 패널 표시 (`main.js:403-418`)
- 포탑 우클릭: 업그레이드 시도 (`main.js:452-471`, `main.js:309-328`)
- 속도 버튼 1x/2x/3x: `setGameSpeed` 호출로 배속 전환 (`index.html:16-18`, `main.js:178-190`)
- 웨이브 입력 + 적용 버튼: 전투 중이 아닐 때 원하는 웨이브로 이동 (`index.html:21-24`, `main.js:218-236`, `main.js:473-491`)
- 스페이스바: 일시정지/재개 (`main.js:504-510`)
- 패배 시 오버레이에서 다시 시작(전체 초기화) 또는 닫기 선택 가능 (`index.html:57-65`, `main.js:331-367`)

## 적 웨이브
- 웨이브당 소환 수: `8 + floor(wave * 1.5)` (`main.js:198-199`, `main.js:541-557`)
- 체력 성장률 `ENEMY_HP_GROWTH_RATE = 1.25`; 이동 속도(`ENEMY_SPEED = 49`)와 보상(`ENEMY_BASE_REWARD = 14`)은 고정 (`main.js:10-13`, `main.js:148-166`, `main.js:524-533`)
- 웨이브 미리보기 패널: 상태(대기/휴식/진행 중/패배), 남은 적 수, 체력, 타일당 초 속도, 보상 표시 (`main.js:154-176`)

## 포탑과 업그레이드
- 기본 사거리 140, 발사 간격 0.55초, 기본 피해 35 (`main.js:437-445`)
- 업그레이드: 골드 소모, 레벨 +1, 피해 1.5배(소수 둘째 자리 반올림), 다음 비용은 두 배; 최대 레벨 7 (`main.js:309-328`, `main.js:17`)
- 포탑 패널: 위치/사거리/발사 간격/피해/레벨/다음 업그레이드 비용 표시 (`index.html:39-46`, `main.js:238-256`)

## 게임 상태 흐름
- `clearCurrentWave()`가 적/발사체를 제거하고 카운터 초기화 (`main.js:196-205`)
- `setWave()`는 게임 오버가 아니라면 원하는 웨이브로 이동하고 미리보기 갱신 (`main.js:218-236`)
- `showDefeatDialog()`와 `resetGame()`이 패배 처리 및 전체 재시작 담당 (`main.js:331-368`)
- `drawState()`는 일시정지/패배 시 오버레이 텍스트 렌더링 (`main.js:771-781`)

## 참고 사항
- 소스에 한글 UI 문자열이 포함되어 있으므로 UTF-8 인코딩을 유지해야 글자가 깨지지 않음
- 자동화된 테스트가 없으므로 변경 후 수동 플레이 테스트 권장
- 기능이나 UI가 크게 바뀔 때마다 `docs/PROJECT_STATE.md`와 본 번역 파일을 함께 갱신할 것
